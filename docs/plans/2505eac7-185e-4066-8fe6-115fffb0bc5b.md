Complexity: Medium

## Goal

Redesign the full frontend into a minimalist, dark-themed Arena experience: a single-purpose landing page showing the arena image, live vote count, shared credit pool, and a single "Generate me an idea" CTA that leads into a streamlined agent brainstorming interaction.

## Context

- **Current state**: The `apps/web` frontend has four routes (`/`=Guide, `/prompt`=PromptStudio, `/vote`=VoteTicker, `/about`=About) with a light slate/indigo theme, multi-tab nav header, CreditsBadge, LangToggle.
- **Backend is correct**: The credit pool (`apps/api/src/services/credits.ts`) is already globally shared (single ledger row). Vote polling (`apps/api/src/jobs/votePoller.ts`) already grants +100 per net-new vote. No backend changes needed except possibly a new SSE-credits endpoint if the landing page needs live credit updates.
- **Reference image**: `docs/requirements/64f6aa2425a2-image.png` — dark cyberpunk/cyborg aesthetic with deep navy/teal palette, neon-like accents, glowing gradient effects. The UI must align with this visual direction.
- **Key constraint**: The existing PromptStudio state machine (INTENT → CONTEXT → CONSTRAINTS → APPROACHES → DESIGN → REFINE → DONE) must remain fully functional; only presentation and navigation change.
- **i18n parity**: All new UI strings need keys in both `en.json` and `zh.json`; existing keys may be reused where applicable.

## Assumptions

- The uploaded reference image at `docs/requirements/64f6aa2425a2-image.png` will be copied into `apps/web/src/assets/arena-hero.png` (or served from `public/`) and used as the hero background on the landing page.
- The backend `/api/votes`, `/api/votes/stream`, `/api/credits`, `/api/prompt/start`, `/api/prompt/answer` endpoints are unchanged and fully working.
- The existing Tailwind setup (PostCSS + `tailwind.config.js`) will be extended with a dark theme palette; no new CSS framework added.
- The simplified navigation does NOT delete the existing Guide, About, or VoteTicker pages — it redirects them or hides them from nav, keeping them in the codebase for future use. A minimal slim header is used instead.
- The landing page vote count polls `/api/votes` every 5 seconds and listens to `/api/votes/stream` SSE for live updates, same pattern as the current VoteTicker page.
- "Shared pool" credit display means showing the balance with a label like "Pool" or "Shared Credits" instead of "Credits"; the backend concept is already correct.

## Acceptance Criteria

- AC-1: **Dark cyberpunk theme applied globally.**
  - `tailwind.config.js` extended with dark color tokens (bg: `#0A0E17`, surface: `#111827`, accent: cyan/teal `#06B6D4`, neon glow gradients).
  - `styles.css` includes global dark background, light text defaults, any neon glow utilities.
  - All existing pages (`PromptStudio`, `NotFound`) restyled to the dark theme; no light-theme leftovers.

- AC-2: **Landing page (`/`) is a single-purpose hero.**
  - Hero section fills viewport with the arena image as background (dark overlay for text readability).
  - Displays the current arena vote count live (fetched from `/api/votes` + SSE stream), styled prominently (large number, glowing accent).
  - Displays the shared credit pool balance from `/api/credits`, labeled clearly as a shared pool (e.g., "Pool: 850 credits remaining — shared by all participants").
  - A single large CTA button: "Generate me an idea" (with a corresponding zh translation). The button links to `/prompt`.
  - No tabs, no sidebar, no complex nav — the entire landing page fits in one viewport.

- AC-3: **Slim minimal header replaces the current complex Header.**
  - New `Header.tsx` contains only: a small brand mark/link (to `/`), the LangToggle, and a small inline credit display (just the pool number + status dot).
  - No multi-tab nav bar. No Guide/Vote/About nav links.
  - The Header must render consistently on the landing page and the PromptStudio page.

- AC-4: **PromptStudio (`/prompt`) streamlined and restyled.**
  - Dark-themed restyle of the existing PromptStudio component — dark cards, cyan/teal buttons, dark textareas.
  - Remove the two-column "Pick a track" vs "Describe your idea" layout.
  - Replace with a single entry: a textarea for free-text idea (optional) and a "Start brainstorming" button. The tracks should be available as quick-select chips/pills below the textarea (clicking one seeds the textarea text and starts, or starts directly).
  - The brainstorming stepper (Q&A conversation) and the final result (generated prompt block with copy button) remain functionally identical but restyled dark.
  - A "← Back" link returns to the landing page.

- AC-5: **Landing page is the root route; PromptStudio is triggered from it.**
  - `App.tsx` routes: `/` → new `Landing` page component; `/prompt` → restyled `PromptStudio`; `*` → restyled `NotFound`.
  - Old pages (`Guide`, `VoteTicker`, `About`) remain in the source tree but are NOT in the route table (or are accessible via hidden routes only).
  - No broken imports — the code still typechecks with `pnpm -r typecheck`.

- AC-6: **i18n complete for all new strings.**
  - New keys added to both `en.json` and `zh.json` for: landing page text (tagline, CTA, pool label, vote label), slim header text, streamlined prompt entry text.
  - Key parity test `apps/web/test/i18n.test.ts` updated to reflect new key sets.
  - No user-facing strings hardcoded outside i18n resources.

- AC-7: **Credit pool global-shared semantics clarified throughout.**
  - Landing page labels the pool as "Shared pool" / "共享积分池".
  - PromptStudio shows pool balance as "Pool: N" instead of "Credits: N" and makes clear deductions come from the shared pool.
  - CreditsBadge (or its replacement) no longer implies per-user balance.

- AC-8: **End-to-end smoke test passes.**
  - `pnpm dev` starts both apps.
  - Landing page loads, shows vote count and pool balance, CTA button is clickable.
  - Clicking CTA navigates to `/prompt`.
  - Starting a brainstorm (free text or track chip) works, credits deduct from shared pool.
  - Completing all steps produces the final CyOps prompt block.
